제약사항 : nuxt 3.6.5, vue 3.3.7, 로 코딩이 되어야하고, 화면에 보이는 텍스트는 모두 영어로 제작되어야해. console log 나 주석은 한글로 작성되어야해.





프로젝트는 부동산 회사에서 엑셀로 관리하던 자산정보를 웹화면에서 관리하는게 목적이야.



화면은 영어로 구성되어 있고, 다국어를 지원할 수 있으면 좋아.



사용자가 보는 메인 화면은 좌측은 지도, 우측은 자산 리스트로 구성되어 있어.



지도에서 자산의 pin을 클릭하거나, 자산리스트(카드형태)에서 자산의 이름을 클릭하면 자산의 모든 정보를 볼 수 있는 상세 화면이 새창으로 열리는 형태야.



사용자 화면의 최 상단에는 검색을 위한 필터링 기능이 메뉴로 구성되어 있어.



이번에는 관리자 화면이야.



관리자화면은 admin 이라는 url 을 통해서 진입하게되.



첫화면에서는 자산을 구별할 수 있는 최소의 정보을 리스트 형태로 보여주고, 수정, 삭제 버튼을 통해서 수정화면으로 이동하거나, 해당 자산을 삭제하게되.



수정화면에서는 자산의 정보가 하나의 화면에 보이게 되고, model 과 거의 1:1 로 매칭된 섹션으로 구성되어 있어. 각 섹션을 수정하면 db 에 해당정보를 업데이트하거나 삭제하고, store 파일에도 수정 내역을 반영해서 자산의 정보를 최신화하도록 구성되어 있어. 셕션을 수정할 때는 우측의 panel 이 화면에 뜨게 되고 panel 에서는 각각의 섹션을 수정하는 form 파일을 로드하게 되.



DB 작업이 이루어질 때는 전체화면 loading 모달을 사용하게 되고, 삭제할 때는 confirm Modal 을 사용하게 되, 이미지를 클릭했을 때나, pdf 파일을 클릭했을 때도 전체 화면으로 modal 을 이용해서 보여주고 있어.



이러한 프로세스들을 관리하는 상태 store 파일이 여러개로 나위어져 있는 상태야.
